# Decision Log

## 2026-05-30 Session

- Built `formally-bmad-ontology-alignment` as a simple workflow because the core flow is linear: select concept, search repositories, compare meaning, propose mapping/import, submit to steward, report.
- Added `scripts/alignment_workspace.py` for deterministic workspace creation, repository registry defaults, mapping skeleton, and report scaffold.
- Did not implement repository API calls in the script. Live search/API use belongs to the workflow layer so retrieved concepts can be interpreted, licensed, and mapped with explicit judgment.
- External ontology concepts enter as `candidate` or `provisional` unless the user explicitly approves promotion.
- The workflow records repository, URI, version/date, license/status, mapping rationale, and source motivation for every proposed import.
- Semantic conflicts block import until repaired, rejected, narrowed, or explicitly overridden through steward policy.
