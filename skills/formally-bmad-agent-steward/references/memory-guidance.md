# Memory Guidance

## Fundamental Boundary

Steward memory informs judgment; the canonical model governs truth.

Use memory for operational learning that improves future stewardship. Use the canonical model and provenance files for anything that affects formal truth, validation, readiness, contradictions, overrides, or auditability.

## Remember

- User preferences for structured English and formal notation exposure.
- Recurring ambiguity patterns and how the owner resolves them.
- Accepted interpretation policies.
- Repair strategies the owner prefers.
- Tool quirks, missing optional backends, and practical limitations.
- Ontology alignment conventions and trusted sources.

## Do Not Remember Here

- Canonical assertions.
- Validation status.
- Provenance links.
- Contradiction or override facts.
- Readiness decisions.
- Tool-run evidence.
- Raw source artifact content.

## Session Logs

Append meaningful direct-maintenance sessions to `sessions/YYYY-MM-DD.md`. Keep entries brief and operational.

```markdown
## Session — {time or context}

**What happened:** {short summary}

**Operational learning:**
- {preference, policy, pattern, or tool quirk}

**Canonical boundary:** {confirm no formal truth was stored only in memory}
```

## Curated Memory

Distill durable operational learning into `MEMORY.md`. Keep it short enough to load every session. Remove stale preferences when the owner changes direction.
